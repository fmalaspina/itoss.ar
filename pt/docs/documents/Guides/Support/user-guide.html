User Guide for ITOSS Support

Using ITOSS in IT Operation

This guide is aimed at IT support people with the goal of showing the functionality of the application and best practices for optimal use.

Accessing the User Interface​

The ITOSS user interface is accessible via a modern web browser compatible with JavaScript, entering the access credentials will be requested.

From a web browser, place the URL of the ITOSS WEB Server and enter the user credentials.

Note: ITOSS has been thoroughly tested with Google Chrome, we recommend using this browser.

Operation Menu​

The operation menu is designed for people responsible for managing and operating components in ITOSS.

When ITOSS is started, it displays component status boards and on the left, the bar to expand the menu. We have the following options:

  • User Key Management.
  • User Preferences.
  • Operation Menu (Operation).

In this menu, the daily activities of review and operational analysis are initiated to understand the overall behavior of the most important operational metrics.

  • Dashboards
  • DE Viewer
  • Life Cycle
  • Explore Menu.
    • Companies
    • Components
    • Reports
  • Configuration Menu.
    • Components
    • Companies
    • Contacts
    • Workgroups

Operational Dashboards​

They provide information about key operational metrics and allow analyzing from a general perspective the indicators of interest.

Status Dashboard​

The general status dashboard (General Status) allows understanding the status of components from different perspectives, by company, by location, and by type of component.

The dashboard allows filtering by environment and by the companies that the support user has under their direct responsibility (Only my companies).

Image of the status dashboard, highlighting in yellow the options for views by different types of grouping.

Image of the status dashboard, highlighting in yellow access to the filter menu by environment and companies responsibility of the connected support user.

Monitor Dashboard​

Monitor dashboards are made up of containers which display the amounts generated according to the analysis performed on the metric. There are public dashboards designed and configured by the ITOSS administrator and custom dashboards by the support user. To access or create new dashboards, select "Add" in the Dashboards screen.

Image showing access to monitor dashboard configuration.

Initially, when entering ITOSS, public dashboards are only displayed if they were assigned by the ITOSS administrator to the support user. If not, it is possible to configure these dashboards to open upon entry.

Image shows the "My dashboards" tab showing the public dashboards assigned to the support user. In the "Public dashboard" tab, the available dashboards are shown.

Image shows how a public dashboard can be opened by default when starting a session in ITOSS.

Creating a Dashboard​

Support users can create their own dashboard with the desired containers for a personalized view.

To create a dashboard, follow these steps:

  1. In the "Dashboard" section, select "Add".

  2. Create a new dashboard by pressing the "+" button.

  3. Enter the name information, label for the new dashboard and indicate that it is a default dashboard. Save the changes with the "Save" button.

  4. The new dashboard will be located in the last tab, position yourself over the dashboard menu to add the desired containers.

  5. Add the containers one by one that will be part of the new dashboard.

  6. Edit the dashboard to configure dashboard display aspects.

Views for Monitor Dashboard​

The dashboards show support staff information on all components, but there is the possibility of creating views on the monitor dashboards to focus on a service, application, location, or other visual requirement.

Views are customized by the support user and configured from the dashboard menu.

The steps to create a new view are:

  1. Add a new view to the desired monitor dashboard by pressing "+" in the dashboard menu.

  2. Configure the view name.

  3. Filter by component, company, location, or workgroup to obtain the desired view. Each entity selection is exclusive in the filter.

  4. Save and close the new view configuration.

  5. Edit an existing view.

Company Dashboard​

Company dashboards are key for the user or support group in a scenario with hundreds of these, it allows quick access to the operational status of the company's components being analyzed.

Access to Company Dashboard​

In the "Explore" exploration menu, there is a section of companies and from there access the control dashboard.

The quickest option is to search by name in the top right of the user interface, placing the company name.

Sections of the Company Dashboard​

The company dashboard is configured by the ITOSS administrator, contains 3 sections/tabs:

  • Dashboard Overview: Initial dashboard with Company information.
  • Component status, with overall totals, by type and environment.
  • DE list

Image of the company dashboard overview.

  • Component List.

Image of the component list on the company dashboard.

  • DE Overview: Information organized by component type and management profile for summarized analysis of the customer's environment behavior.

Image of the analysis of the status and DE of all the company's components grouped by type and management profile.

Monitor Reports and Dashboards​

From the menu available on the company dashboard, we can access to view or create monitor dashboards and analytical reports.

The monitor dashboards limit the information presented to the components of this company, public monitor dashboards can be viewed or custom dashboards can be created with the components and containers defined by the support user.

Reports are managed by the ITOSS administrator and are available to run on demand from the client interface.

Image of the available reports for Lumen company.

Component Dashboard​

The component is the last link in the analysis, where more information is required and there is more interaction with support users. The component dashboard not only allows us to analyze operational and support information of the component, but also gives us tools for a simpler and more efficient operation.

The component dashboard can be accessed from the general search, in the Explore menu in the "Components" section, and from a DE present in some of the other previously mentioned dashboards.

Component Dashboard

The component is the final link in the analysis, where more information depth is required and there is more interaction with support users. The component dashboard not only allows us to analyze operational and support information for the component, but it also gives us tools for simpler and more efficient operation.

Access to the Component Dashboard

The component dashboard can be accessed from the general search, in the Explore menu in the "Components" section, and from a DE present in some of the other previously mentioned dashboards.

Image of the component explorer, allows filtering and accessing the component dashboard from the "eye" design icon.

Image to access the component dashboard from the quick search in the upper right corner of the user interface.

Image of the DE list in a container in the monitor dashboard, allowing easy access to components with abnormal situations.

Sections of the Component Dashboard​

The component dashboard has 3 initial sections, and the rest of the sections will be related to the management profile assigned to the component.

The sections are:

  • Component:
    • General component information.
    • Related components.
    • Status of monitors.
    • List of component DEs.
  • History: information related to the component's support management, such as changes in the lifecycle, user notes, and DE management.
  • Operational Log: information related to diagnostic and/or remediation tasks.
Component Section​

This is the initial section when entering the component dashboard, and it provides general information, relationships with other components, component DEs, and monitor status.

This section displays a bar with actions that allow support users to streamline the component's operational management.

Action bar allows the following management activities:

  • Add a note to the component (displayed in the history).

  • Monitor execution (corresponding to the component's management profile). It is possible to run the status monitor, a selection of monitors, or all monitors.

  • Lifecycle management (*): schedule maintenance.

  • Lifecycle management (*): take the component out of service.

  • Connection parameter adjustments for the component.

  • Management reset. This activity eliminates component status information and all its DEs, starting management from scratch again. IT IS NOT RECOMMENDED to perform this action, only in very specific cases that require it.

  • Detailed information about the component's management profile.

  • Status of the last execution of the monitors.

(*) Actions that perform changes in lifecycles will vary in the bar according to the state of the component, this is the case for a component in operational state.

Component types that have defined tools add an action to the bar.

  • Tool. Diagnostic and remediation tools to be executed on the component on demand.
History section​

It is a log that records operational management activities carried out by support users, such as changes in the life cycle, user notes, DE management, among others. The history is not recycled over time.

Operational Log section​

It is a log for recording diagnostic and/or remediation information.

Other sections​

The information available depends on the management profile associated with the component, therefore the other sections are adjusted to the profile.

In other sections, data is generally displayed referring to performance, storage, and configuration for a total analysis of the component's key metrics. Some examples.

DE and Notification Management​

DE and notification management is key for an efficient operation of the component, improving the availability and quality of IT services.

DE Management​

DEs can be viewed on the operational boards and in the DE viewer in the Operational menu.

Image of the "Dashboard entry viewer" that allows exploring DEs from all components.

The support user can "attend" a DE with the objective of indicating that work is being done or information is being provided about the situation.

Image highlighting the icon that allows "attending" a DE.

Notification Management​

Notifications are directed to support users for considered urgent situations, they can be deleted (it only has an effect for the connected support user) individually or globally.

The "bell" style icon in the top right corner indicates the number of notifications and is the access to the notifications list.

Image showing the icon to access notifications in ITOSS.

Image shows the icon to delete individually and globally.

From a notification, you can access the component dashboard, which is very useful for quickly validating the current situation of the component.

In the preferences menu, the reception or non-reception of notifications can be adjusted, and the possibility of triggering a sound when a notification arrives.

Component Life Cycle Management​

In the Operations menu, there is a section for managing the life cycle of components (Component life cycle), from this site it is possible to make massive status changes, test status and monitors.

Reports​

In the "Report" section of the "Explore" menu, there are ITOSS on-demand reports, which are prepared and published by the application administrators.

Reports are grouped by component type and allow generating information from the entire environment, by customer, or location.
